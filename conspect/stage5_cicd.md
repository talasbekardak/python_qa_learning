# Этап 5: CI/CD — GitHub Actions

## Что такое CI/CD?

**CI (Continuous Integration)** — автоматический запуск тестов каждый раз когда кто-то пушит код на GitHub. Не только перед продакшном, а при каждом изменении.

**CD (Continuous Delivery/Deployment)** — автоматическая доставка кода на сервер после того как тесты прошли.

> **Аналогия:** Пекарня. Каждый раз когда пекарь делает новый хлеб:
> - CI — автоматически проверяет вкус, запах, вид (тесты)
> - CD — если всё ок, автоматически везёт хлеб в магазин (деплой)

В нашем проекте мы делаем только **CI** — автозапуск тестов при каждом пуше на GitHub.

---

## Что такое Workflow файл?

Workflow файл — это **инструкция для GitHub**: "когда кто-то пушит код, сделай вот это автоматически".

Это просто список команд, которые ты бы сама вводила в терминале — только записанные в файл.

Файл живёт по пути: `.github/workflows/название.yml`

Формат файла — **YAML** (отступы важны!).

---

## Структура Workflow файла

```yaml
name: UI Tests          # название — видно на GitHub во вкладке Actions

on:                     # КОГДА запускать?
  push:
    branches: [ main ]  # при каждом пуше в ветку main

jobs:
  test:                 # название задачи (можно любое)
    runs-on: ubuntu-latest  # на каком компьютере запускать (виртуальный, бесплатный)

    steps:              # ШАГИ — что делать по порядку
      - name: Описание шага
        uses: ...       # готовый инструмент из библиотеки GitHub
        # ИЛИ
        run: команда    # обычная команда в терминале
```

### Ключевые блоки:
| Блок | Что означает |
|------|-------------|
| `name` | Название workflow |
| `on` | Событие-триггер (когда запускать) |
| `jobs` | Список задач |
| `runs-on` | ОС виртуального компьютера |
| `steps` | Шаги внутри задачи |
| `uses` | Готовый action из маркетплейса GitHub |
| `run` | Команда в терминале |
